# 디자인 의사결정 기록

## 작성 기준

- 최종 결과물에 영향을 준 디자인 판단만 기록합니다.
- 색상, 화면 구성, 정보 우선순위, 버튼 위치, 이미지 스타일 등 변경 이유가 필요한 내용을 남깁니다.
- 관련 Drive 링크 또는 Figma 링크가 있으면 함께 기록합니다.

## 최종 제출 대상

- 프로젝트명: 냉장고 파먹기 / fridge-chef
- 서비스 목표: 냉장고 속 보유 재료를 기반으로 만들 수 있는 레시피를 추천하고, 부족한 재료를 장보기 행동으로 연결합니다.
- 화면 구성: 메인 화면 -> 레시피 목록 화면 -> 레시피 상세 화면
- 최종 UI 이미지:
  - [메인 화면](../assets/final-ui/01-main-screen.jpg)
  - [목록 화면](../assets/final-ui/02-list-screen.jpg)
  - [상세 화면](../assets/final-ui/03-detail-screen.jpg)

## 메인 화면 의사결정

메인 화면은 사용자가 가장 먼저 해야 하는 행동인 재료 입력을 중심에 둡니다. 초기 프롬프트에는 검색바와 재료 입력 영역이 함께 있었지만, 두 요소가 모두 탐색/입력 역할을 맡으면 첫 행동이 흐려질 수 있습니다. 따라서 재료 칩, 추가 버튼, 재료 카운터, Find Recipes 버튼을 중심으로 사용자의 시작 행동을 명확히 하는 방향을 기록했습니다.

## 목록 화면 의사결정

목록 화면은 추천 레시피를 비교하는 공간입니다. 추천 레시피 카드는 음식 이미지, 레시피명, 조리 시간, 난이도, 재료 일치율, 보유 재료 근거 문구를 포함하도록 설계했습니다.

정보 우선순위는 다음과 같습니다.

1. 레시피명과 음식 이미지: 사용자가 무엇을 만들지 빠르게 인지합니다.
2. 조리 시간과 난이도: 자취생의 현실적인 선택 기준입니다.
3. 재료 일치율과 보유 재료 근거: 추천 결과를 신뢰하게 하는 기준입니다.
4. 필터 칩: 빠른 조리, 채식, 15분 이내처럼 비교 조건을 좁힙니다.

## 상세 화면 의사결정

상세 화면은 선택한 레시피를 실제 조리로 이어가는 화면입니다. 완성 요리 이미지를 상단에 배치해 기대 결과를 먼저 보여주고, 레시피명, 조리 시간, 난이도, 1인분, 칼로리 정보를 바로 아래에 둡니다.

그 다음 재료와 부족 재료를 확인하게 하고, 부족 재료 옆에 장보기 연결 행동을 둡니다. 조리 단계는 번호형 단계로 정리해 사용자가 화면을 보며 바로 따라 할 수 있게 했습니다. 하단의 Start Cooking 버튼은 상세 정보를 확인한 뒤 실행으로 넘어가는 주 행동입니다.

## 색상/톤앤매너

- 기본 배경: 화이트
- 포인트 컬러: 소프트 그린
- 텍스트: 다크 그레이
- 부족 재료 표시: 라이트 오렌지

화이트 + 소프트 그린 톤은 음식/생활 앱에서 가볍고 위생적인 인상을 주며, CTA 버튼과 재료 칩을 부담 없이 강조할 수 있습니다. 미니멀 UI를 선택한 이유는 사용자가 재료, 추천 결과, 조리 단계라는 핵심 정보에 집중해야 하기 때문입니다.

## 일관성 유지 전략

- 화면비: 9:16 모바일 화면비 유지
- 카드: 둥근 카드와 넉넉한 여백 반복
- 버튼: 주요 행동은 소프트 그린 CTA로 통일
- 색상: `#FFFFFF`, `#A8D5BA`, `#333333`, `#F5C396`처럼 구체 색상 기준 기록
- 타이포그래피: consistent sans-serif typography 조건 유지
- 여백: 미니멀 UI에 맞게 화면 요소 사이를 넉넉히 둠
- 흐름: 메인 -> 목록 -> 상세의 3단계 구조 유지

이 전략은 화면마다 역할은 다르지만 같은 서비스의 연속 화면으로 보이게 하기 위한 기준입니다.

## 후가공 판단

Figma에서 최종 화면을 배치하고 투명 Hotspot으로 화면 흐름을 연결하는 방식은 이미지 3장이 단순 산출물에 머무르지 않고 사용자 플로우로 평가될 수 있게 하기 위한 선택입니다. 또한 최종 검수에서 텍스트 가독성, UI 정렬, 화면 간 색상/카드/버튼 일관성을 확인하도록 정리했습니다.

외부 Figma 링크의 실제 내용을 직접 열람했다고 가정하지 않으며, repo 내부 문서에 기록된 Figma 프로토타입 링크와 작업 로그를 근거로 작성했습니다.
