# 작업 운영 가이드

## GitHub에서 작업하는 원칙

- GitHub는 작업 과정과 문서를 관리하는 공간입니다.
- 원본 이미지, 대용량 이미지, PDF 원본은 GitHub에 올리지 않습니다.
- 모든 작업은 가능한 한 Issue 단위로 나누어 진행합니다.
- 최종 산출물 원본은 Google Drive에 저장하고, GitHub에는 링크만 기록합니다.

## Issue 단위로 작업을 나누는 방법

1. 작업 목적이 명확한 단위로 Issue를 만듭니다.
2. 담당자, 입력자료, 산출물, 완료 기준을 작성합니다.
3. 작업 중간 결과와 Drive 링크를 댓글에 남깁니다.
4. 검수 요청이 필요하면 review-request 템플릿을 사용합니다.
5. 수정이 끝나면 최종 링크를 관련 문서에 반영하고 Issue를 닫습니다.

## Google Drive에 결과물을 올리는 방법

- 폴더 구조는 [google-drive-structure.md](google-drive-structure.md)를 따릅니다.
- 원본, 후가공본, 최종본을 섞어 저장하지 않습니다.
- 최종 제출용 파일은 `99_최종제출/` 폴더에 모읍니다.

## Drive 링크 기록 방법

- 각 Issue 댓글에 작업 결과물 Drive 링크를 남깁니다.
- 최종 채택된 파일은 [final-submission-links.md](final-submission-links.md)에 기록합니다.
- 프롬프트 단계별 결과물은 [prompt-log.md](prompt-log.md)에 링크를 남깁니다.

## 파일명 규칙

```text
ProjectA_화면명_v번호_담당자_날짜.확장자
예: ProjectA_Main_v01_KBJ_0624.png
```

- 화면명은 Main, List, Detail처럼 역할이 드러나게 작성합니다.
- 버전은 v01, v02처럼 2자리 숫자로 관리합니다.
- 담당자는 팀에서 합의한 이니셜을 사용합니다.
- 날짜는 MMDD 형식으로 작성합니다.

## 이미지 원본을 GitHub에 올리지 않는 이유

- 이미지와 PDF는 용량이 커져 저장소 관리가 어려워질 수 있습니다.
- 최종 산출물은 Drive에서 버전과 공유 권한을 관리하는 편이 안전합니다.
- GitHub에는 문서, 링크, 체크리스트만 두면 작업 흐름을 추적하기 쉽습니다.

## 작업 완료 기준

- 산출물이 Google Drive에 업로드되어 있습니다.
- Issue 댓글에 결과물 링크가 있습니다.
- 필요한 문서에 최종 링크가 반영되어 있습니다.
- 검수자가 과제 요구사항 충족 여부를 확인했습니다.

## 팀장 검수 흐름

1. 담당자가 Issue에 검수 요청 댓글을 남깁니다.
2. 팀장 또는 검수자가 Drive 파일과 문서를 확인합니다.
3. 수정 필요 사항을 Issue 댓글로 남깁니다.
4. 담당자는 수정본을 업로드하고 링크를 갱신합니다.
5. 검수자가 최종 승인 후 Issue를 닫습니다.
